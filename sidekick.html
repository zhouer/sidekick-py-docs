sidekick.base_module module

Base Class for Sidekick Visual Modules.

This module provides the foundational BaseModule class, which acts as a blueprint for all specific visual modules in the Sidekick library (like Grid, Console, etc.).

You typically won’t use BaseModule directly in your scripts. Instead, you’ll use the specific module classes (e.g., sidekick.Grid). This base class handles the common tasks needed for any visual module:

  • Assigning a unique ID to each module instance.

  • Ensuring the connection to the Sidekick panel is established.

  • Sending commands (like ‘create’, ‘update’, ‘remove’) to Sidekick.

  • Providing a way to remove the visual element (remove() method).

  • Handling errors reported by Sidekick for a specific module (on_error callback).

sidekick.connection module

Sidekick WebSocket Connection Management.

This module is the “engine room” of the Sidekick Python library. It handles all the behind-the-scenes communication with the Sidekick visualization panel, usually via a WebSocket connection.

As a user of the sidekick library, you typically don’t need to interact with this module directly. Most of its important functions (like set_url, run_forever, shutdown, ensure_ready, flush_messages) are made available directly under the main sidekick package (e.g., sidekick.set_url()).

Key responsibilities managed here include:

  • Establishing and maintaining the WebSocket connection.

  • Running a background thread to listen for messages from Sidekick.

  • Sending messages (commands) from your Python script to Sidekick.

  • Buffering messages if the connection isn’t ready yet.

  • Handling system-level messages for discovering peers (Hero/Sidekick).

  • Managing the connection state (Disconnected, Connecting, Ready).

  • Providing functions to control the script’s lifecycle (run_forever) and ensure messages are sent (flush_messages).

  • Cleaning up the connection gracefully (shutdown).

sidekick.connection.run_forever()[source]

Keeps the Python script running indefinitely to listen for Sidekick events.

If your script needs to react to user input from Sidekick (like button clicks via Control.on_click or text input via Console.on_input_text), you should call sidekick.run_forever() at the end of your script setup.

This function will block the script from exiting, allowing the background thread to continue receiving messages and triggering your callback functions.

To stop the script gracefully, either: 1. Press Ctrl+C in the terminal. 2. Call sidekick.shutdown() from within one of your event callback functions.

Examples

import sidekick
import time

console = sidekick.Console(show_input=True)
control = sidekick.Control()

def handle_input(text):
    console.print(f"You entered: {text}")
    if text == "quit":
        console.print("Shutting down...")
        sidekick.shutdown() # Stop run_forever

def handle_click(control_id):
    console.print(f"Button '{control_id}' clicked!")

console.on_input_text(handle_input)
control.on_click(handle_click)

control.add_button("my_button", "Click Me")
console.print("Enter text below or click the button.")
console.print("Type 'quit' to exit.")

# Keep the script running to handle events
sidekick.run_forever()

print("Script finished.") # This line will run after shutdown() is called
Returns:

None

sidekick.connection.flush_messages(timeout: float | None = None) → bool[source]

Waits until the connection is READY and all buffered messages have been sent.

When you send commands (like grid.set_color()) before Sidekick is fully connected and ready, the library temporarily stores these messages in a buffer. This function waits until the connection is established (CONNECTED_READY) AND all messages in that buffer have been sent to Sidekick.

This is particularly useful at the very end of short scripts that don’t use run_forever(). Calling flush_messages() increases the chance that Sidekick receives and displays all your updates before the script exits.

Parameters:

timeout (Optional[float]) – Maximum time in seconds to wait. If None (default), waits indefinitely. If the timeout is reached before the buffer is flushed while ready, returns False.

Returns:

True if the connection was ready and the buffer was flushed within

the timeout, False otherwise (e.g., timeout, disconnection).

Return type:

bool

Examples

import sidekick
import time

# Create a grid and update cells quickly
grid = sidekick.Grid(4, 4)
for i in range(4):
    grid.set_color(i, i, color="blue")
    time.sleep(0.01) # Simulate some work

print("Script finished sending commands.")

# Wait up to 5 seconds for messages to be sent before exiting
if sidekick.flush_messages(timeout=5.0):
    print("Messages flushed successfully.")
else:
    print("Warning: Messages might not have been fully sent.")

# No run_forever(), so script exits here. flush_messages helps ensure
# the blue cells appear in Sidekick.
